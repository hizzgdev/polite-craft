# 产品详情
(请着重说明该内容的用途以及用户为何应该安装它)

如果你经常在 GitHub 上参与 PR Review、Issue 讨论，却总要经历这样的流程：

脑子里用中文想好怎么说 → 打开 ChatGPT → 粘贴上下文和草稿 → 复制英文 → 贴回 GitHub

GitHub 得体英文 就是为此而生的 Chrome 扩展。你在评论框里直接写中文，点一下按钮，扩展会自动读取当前 PR/Issue 的上下文（标题、描述、最近对话），结合你选择的语气和与对方的关系，生成得体、可直接发送的英文回复，并填回输入框。

## 它能帮你做什么？

  • 写中文，出英文：在 GitHub PR / Issue 评论框写中文草稿，一键生成英文
  • 语气可控：建设性建议、温和鼓励、直接反馈、正式汇报，四种风格任选
  • 关系可控：社区贡献者、同事、熟人、上级，英文措辞会随场景调整
  • 懂上下文：自动抓取页面信息，回复更贴题，而不是生硬直译
  • 保留原文：生成结果在原文下方追加英文，方便对照修改

## 你为什么应该安装它？

1. 省时间，少打断

不必在 GitHub 和 AI 工具之间来回切换、复制粘贴。评论框下方就是操作栏，写完中文、选好语气，点一次即可。

2. 英文更「得体」，而不只是「能看懂」

开源协作里，同样一句意思，对 contributor 和对 maintainer 的写法应不同。扩展按语气与关系生成英文，减少过于生硬、过于随意或过于「中式客套」的表达。

3. 为中文开发者设计

界面与选项都是中文，默认面向 GitHub 协作场景，无需把产品当「给英语母语者用的工具」来理解。

4. 隐私可控

DeepSeek API Key 只存在本机；对话里的参与者以 current user / user1 等匿名标签发给 AI，不会附带 GitHub 用户名映射。

适合谁？

  • 在 GitHub 上用英文 Review、讨论 Issue 的中文开发者
  • 能写中文思路，但希望英文回复更专业、更自然的人
  • 想减少「写一条评论要切好几个窗口」的人

如何使用？

  1. 安装扩展，在设置里填入 DeepSeek API Key
  2. 打开任意 GitHub PR 或 Issue 页面
  3. 在评论框写中文，选择语气和关系，点击 Generate English
  4. 检查生成的英文，稍作修改后发送

一句话：它是挂在 GitHub 评论框旁的英文写作助手——你用中文思考，它帮你写出得体、可直接发送的英文。



## 单一用途说明

GitHub 得体英文 只做一件事：当你在 GitHub 的 Pull Request 或 Issue 页面撰写评论时，辅助你把中文思路写成合适的英文，以便参与英文协作讨论。扩展不会在 GitHub 以外的网站运行，也不提供网页全文翻译、广告、新闻聚合、密码管理或与上述功能无关的其他能力。

具体工作方式如下： 扩展仅在 github.com 的 PR / Issue 评论输入框下方显示操作栏。你在评论框中输入中文草稿，选择语气（如建设性建议、温和鼓励、直接反馈、正式汇报）以及与对方的关系（如社区贡献者、同事、上级），点击「Generate English」后，扩展会读取当前页面中与该评论相关的公开信息（如 PR / Issue 标题、描述、最近对话摘要），连同你的中文草稿一并发送给你自行配置的 DeepSeek API，生成英文回复并写回评论框。你可在发送前自行修改生成结果。

数据使用范围严格限于实现上述功能： 扩展仅在用户主动点击生成时读取当前 GitHub 页面上的相关内容，以及用户输入的中文草稿；DeepSeek API Key 与偏好设置保存在用户本地浏览器中。扩展不会收集、出售或用于广告投放；不会将数据用于与「GitHub 英文评论写作辅助」无关的目的；不会代替用户自动发送评论。

使用前提： 用户需自行申请并填写 DeepSeek API Key。没有 API Key 时，扩展无法完成英文生成，但上述单一用途不变。

## 需请求 storage 权限的理由

本扩展请求 `storage` 权限，仅用于在用户的本地浏览器中保存与核心功能直接相关的少量配置，不涉及浏览历史、评论内容或翻译结果的持久化存储。

**第一，保存 DeepSeek API Key。** 扩展需调用用户自行申请的 DeepSeek API 才能将中文草稿生成英文。API Key 由用户在设置页主动填写并保存；翻译时由后台脚本从本地读取，用于向 `api.deepseek.com` 发起请求。Key 仅保存在 `chrome.storage.local`，不上传至开发者服务器，也不写入网页 DOM。

**第二，记住用户的语气与关系偏好。** 在 GitHub 评论框下方的工具栏中，用户可选择语气（如建设性建议、直接反馈）及与对方的关系（如社区贡献者、同事）。扩展会将用户最近一次选择保存在本地，以便下次打开 GitHub 页面时自动恢复，避免每次重复设置。用户修改下拉选项时会即时更新本地保存值。

**第三，提供首次安装的默认配置。** 扩展首次安装时，会在本地写入默认语气与关系（建设性建议、社区贡献者），确保用户首次使用时有合理初始选项。

上述数据均通过 `chrome.storage.local` 读写，仅在本扩展内部使用，不用于广告、统计分析、用户画像或与「GitHub 英文评论写作辅助」无关的功能。未使用 `chrome.storage.sync`，配置不会随 Google 账号同步到其他设备。除 API Key 与两项偏好外，扩展不通过 storage 保存任何其他信息；页面上下文与翻译内容仅在用户点击生成时临时使用，不会写入 storage。

因此，`storage` 权限是本扩展实现「保存用户配置、持续提供翻译服务」所必需的最小权限，范围清晰、用途单一，且与用户可见的设置功能一致。

## 需请求主机权限的理由

本扩展在 manifest 中声明以下主机权限（host permissions），均与单一用途直接相关，且已尽量收窄范围：

**1. `https://github.com/*`**

用于在 GitHub 的 Pull Request / Issue 页面注入 content script（`content.js`、`content.css`），在评论输入框下方显示操作栏；并在用户点击「Generate English」时，从当前页面 DOM 读取与评论相关的公开信息（如标题、描述、最近对话摘要），作为生成英文的上下文。扩展仅访问 GitHub 网站，不访问其他域名。

**2. `https://api.deepseek.com/*`**

用于将用户的中文草稿及页面上下文发送至用户自行配置的 DeepSeek API，以获取英文翻译结果；同时在用户于设置页保存 API Key 时，发起一次轻量级请求以校验 Key 是否有效。除 DeepSeek 官方 API 外，扩展不向其他第三方服务器发送用户数据。

未申请 `http://`、通配全站或其他与上述功能无关的主机权限。content_scripts 的 `matches` 与 host_permissions 中的 GitHub 匹配模式一致，均为 `https://github.com/*`。

## 项目中使用远程代码吗？如果使用，请说明理由

**不使用远程代码。**

本扩展的全部 JavaScript 均打包在扩展程序内，包括 `background.js`、`content.js`、`popup.js` 等，不通过 `<script src="...">` 引用外部 JS 文件，不使用 `eval()` 或 `new Function()` 执行远程传入的代码字符串，也不加载外部 WebAssembly 模块。

扩展在运行时会向 `https://api.deepseek.com` 发起 HTTPS 网络请求，但仅传输用户草稿、页面上下文等文本数据，并接收 JSON 格式的 API 响应；返回内容仅作为文本结果写回评论框，**不会在浏览器中作为代码下载或执行**。因此，这不属于 Chrome Web Store 所定义的「远程代码」。

综上，本扩展符合「所有可执行逻辑均包含在提交包内」的要求，不依赖远程代码实现核心功能。
